STRANDED

Personal Project


Description

"Stranded" is a Roblox horror and adventure game that I started working on back in 2020. I have always been a fan of everything horror. The suspense, the music, the thrill, the mystery, the characters, all of it was fascinating to me. Everything about the horror genre motivated me to make something that would allow players to experience something new. I wanted people to understand the passion I have for things that make a genre like horror unique.

Before Stranded, I had some experience working on other projects on Roblox (some of which can be seen here). However, I felt that nothing I had made or been a part of really showcased my skills as a builder, coder, designer, and storyteller. I wanted to create something original, exciting, and entirely from the ground up. So, I abandoned everything I had worked on up to that point. I wanted to create a game that told a story. As time went on, I took the opportunity to turn this project into a learning experience. Going into the development of Stranded, I did not know how to code.

As of 2024, I have been able to learn a lot that the Lua programming language has to offer. I have practiced my building and design skills over and over, and have been extremely grateful for the platform that Roblox has given me to be able to create something that I can be proud of. I have taken a lot of inspiration from other games and movies for this project. However, the game is still a work in progress and nothing on this page is final.

I began working on Stranded before generative AI tools became widely used. My main motivation for this project was to challenge myself and see what I could create, not to have something generate my ideas for me. I believe that AI can be a useful tool for things like programming and processing data, but I do not think that people should rely on it. AI lacks the depth of emotional understanding and subjective interpretation that people bring to their creative ideas.

INTRO

The intro to Stranded is the very first cinematic I designed for the game. I wanted to create a cutscene that was synced to a soundtrack. I began working on it very early, probably going through it more than 100 times to make sure everything was exactly how I wanted it. Every character animation was made using the Live Animation Creator feature in Roblox Studio. I believe the most important part of a game is how it begins. When a player first joins the game, I wanted to grab their attention by having them directly involved in the story. Every player sees their own character in each of the cutscenes.

CODE

--IF PLAYED, THEN
if ValuesData["Progress"]["CompletedIntro1"] == true then
	script.ReplayIntro.Value = true

	CutscenesModule.Menu(Player)

	task.wait(2)
	script.Enabled = false

--IF HAVEN'T PLAYED, THEN
elseif ValuesData["Progress"]["CompletedIntro1"] == false then

	CutscenesModule.Intro1(Player)

	task.wait(2)
	script.Enabled = false

end

When someone joins the game for the first time, the intro plays before they are redirected to the game menu. The code above checks to see if a player has completed the intro, and then calls onto a module known as the Cutscenes module to run a specific function. If a player has played before, they are redirected to the menu with an option that allows them to replay the intro. If a player hasn't played before, they must first complete the intro before they can be brought to the menu. After completing the intro, the CompletedIntro1 value is then set to true and saved by a DataStore.

MENU

I wanted to give Stranded's menu a lot of background detail. Players are able to go straight into the lobby by pressing "New Game" or "Continue", gain some information about the game by pressing "Information", see the game credits by pressing "Credits", or replay the intro by pressing "Replay Intro". There are a total of 15 songs that each have a chance of playing every time a player visits the menu. The lack of color in the menu is intentional. Every player sees their own character sitting on the hill.

local module = {}

return module

Here is a basic setup of how the Cutscenes module is structured. Each of the variables are accessed from both module functions, as both the menu and intro deal with elements from Players, Lighting, TweenService, UserInputService, ReplicatedStorage, and Debris. The CameraShaker and GlobalFunctions modules are accessed as well. The GlobalFunctions module is a module that I created that takes care of disabling a player's movements, resetting the game's lighting, muting specific sounds, and more. While both module functions have a considerable amount of code in each, the intro has a lot more. The amount of code required in a cutscene module function depends on the length of the cutscene.

MENU FUNCTION

function module.Menu(Player)

	local Character = Player.Character
	local HumanoidRootPart = Character:WaitForChild("HumanoidRootPart")
	local Humanoid = Character:WaitForChild("Humanoid")

	local BHS = Humanoid:WaitForChild("BodyHeightScale")
	local BTS = Humanoid:WaitForChild("BodyTypeScale")
	local BPS = Humanoid:WaitForChild("BodyProportionScale")

	local Height = BHS.Value*(4 + BTS.Value*math.pi/2*(1 - 1.2/math.pi*BPS.Value)) + 1

	HumanoidRootPart.Anchored = true
	Character.Archivable = true

	--IF TALLER THAN DEFAULT, THEN
	if Height > Default then

		local NewHeight = Height - Default

		--RAISE FIXCHECK HEIGHT
		local NewYPosition = FixChecks.MenuFixCheck.Position.Y + NewHeight
		FixChecks.MenuFixCheck.Position = Vector3.new(X, NewYPosition, Z)

	--IF SHORTER THAN DEFAULT, THEN
	elseif Height < Default then

		local NewHeight = Height - Default

		--LOWER FIXCHECK HEIGHT
		local NewYPosition = FixChecks.MenuFixCheck.Position.Y - NewHeight
		FixChecks.MenuFixCheck.Position = Vector3.new(X, NewYPosition, Z)

	end

	...

end

Ever since Roblox allowed users to edit their own avatar's height and scale, one of the greatest challenges I faced when creating this game was determining a player's height. Because I was using a very specific position for a character's model in each cutscene of the intro (which I label as a FixCheck), a player's character would sometimes be too low in the ground for being too tall or too high above the ground for being too short. In the Cutscenes module, there is a variable "Default" that has a very specific number assigned to it. This number is the exact height of a default block avatar rig. In both of the module functions, a formula is used to calculate a player's height based on their BodyHeightScale, BodyTypeScale, and BodyProportionScale. Then, each cutscene position (FixCheck) is adjusted based on whether the player's avatar is taller or shorter than the default value. The code above is an example of how this process is done for the player's character position in the menu.

LEVEL DESIGN

THE AGENCY (LOBBY)

❮
❯
1/58

The Agency is one of the biggest and most complex builds that I have ever made by myself. I took heavy inspiration from Dev Rel Development's 8th Annual Bloxy Awards. The Agency consists of multiple sectors, including the Main Hall, Artifact Gallery, Entity Chambers, Offices, Corridors, and the Core. I have spent many years putting this lobby together, and it is arguably one of my best builds.

❮
❯
1/5
As you can see, there are many similarities between the lobby I designed and the lobby I took inspiration from. I wanted to expand on this design and make The Agency extremely large in order to encourage players to explore. I have always been a fan of adventure and exploration games on Roblox, especially the 2017 and 2018 Egg Hunts.

There are a total of 35 lobby soundtracks that each have a random chance of playing in rotation once a player enters the lobby. When music plays, the speakers around the facility pulse to the beat of whichever soundtrack is playing. Similar to Dev Rel Development's lobby, announcements with random hints play over the lobby intercom, while also muffling any background music while being played. Coins are scattered throughout many of the different sectors, rewarding those who enjoy exploring. There are many secrets to find and quests to complete in the Agency. Randomly-generated NPCs that players can interact with also roam around the lobby, allowing players to also complete side quests.

CHARACTER DESIGN

STRANDED

The game's antagonist. A powerful sorcerer with an unknown origin. Almost every detail about this individual is either missing or redacted. His real name is unknown and his motives are unclear. Goes by the alias "Stranded" and is known to be the primary enemy actively working against The Agency.

The game's title comes from this character, which was made after I found out that I had made an alternate account on Roblox by the username of StrandedBeneath. I had no memory of creating this account so I was surprised to find that I had registered it on the exact same date as my main account. The username of this account is what first gave me the idea to start working on my own horror game. More on the early development of this project later.

TREASURE

The antagonist's main accomplice. Unknown origin. Similar to his partner, almost every detail of this individual is either missing or redacted. His real name is unknown and his motives are unclear. Goes by the alias "Treasure" and is also known to be actively working against The Agency.

While on a search for other accounts that I may have registered to Roblox in the past, I came across one that I made with the username TreasureTechnique. Like StrandedBeneath, I had no memory of creating this account. Interestingly, I made this account a few months before the other two. I liked the username enough to want to have him be involved in the story.

AGENT ZEN

Leader of The Agency. Agent Zen's ultimate goal is to put an end to Stranded's antagonism by containing all potentially dangerous entities from other worlds before Stranded can use them to cause irreversible damage to his world.

This character was inspired by my main Roblox account. I have always been interested in things like secret agents and espionage, so I wanted my character to resemble something similar to those themes. I also wanted to have a character that could lead players and explain their objective to them. A lot of horror games that I have played on Roblox don't give a whole lot of context or details on what to do or how to get to the next level, which can be frustrating. Agent Zen is going to be one of the more important characters throughout the story.

ENTITY: AZALEA

Queen of the Woodlands. Once a divine being of pristine elegance, Azalea was the marked entity that reigned over the mystical lands of Aezamund. Despite her captivating and enchanting facade, her veil of corruption is often concealed.

Queen Azalea is one of my first original characters that I ever made. I wanted to give her a sort of mother nature vibe considering the landscape she would reside in. It was exciting being able to experiment with character design in Roblox Studio because it was something I had not tried before. She is definitely one of the favorite entities among my friends and I, and I am excited to see what people will think of her in the game.

ENTITY: THE WATCHER

Knight of Iakudell. Cursed to only observe those who enter his realm, The Watcher was the only surviving knight in the kingdom of Iakudell.

The Watcher is definitely the entity I am most proud of. I have redesigned him multiple times, with this final design being my favorite. Iakudell is a very desert-like realm, so I wanted the entity of this dimension to resemble a deceased pharaoh or emperor. His design as well as his story is heavily inspired by "The Chaser" from the Saint Abida Tribe's Realm of the 9 Portals.

ENTITY: OMNIS

Celestial deity of Orion. Forever confined to the expanse of his realm, Omnis was banished to Orion for unknown reasons. A storyteller by heart, he enjoys narrating stories to those that find themselves before him.

When I began working on Stranded, I knew that I wanted to create a character that could explain what is going on to a player. Whether it was an event that occurred, a character's backstory, or explaining how a player died, I knew I did not want to leave players feeling confused. I thought that having a character act as a narrator throughout the game could be pretty beneficial.

ENTITY: VORATH

Lost soul of Whitmore. Summoned as a malevolent angel of shadows and whispers, Vorath was bound to the Whitmore Manor through a dark pact. His name strikes fear into the hearts of those who dare speak it, a reminder of the darkness that lurks within every soul.

The design for Vorath comes from one of my good friends, Charlie. He is a phenomenal artist and actually offered to draw this entity for Stranded. You can view his original drawing by clicking on the image above. I have plans to rework Vorath's story and in-game design, hoping to make him look a little more accurate to his drawing in the future.

USER INTERFACE

LOADING SCREEN

The loading screen is the first thing that a player sees after joining the experience. After preloading some of the more essential content, the loading screen displays some in-game screenshots, a few useful tips, the amount of assets loaded, and a loading mask icon. The in-game screenshots as well as the tips are randomized each time a player is brought to the loading screen. However, I have plans to give a lot of the current UI a full makeover, especially the loading screen.

HUB GUI

The Hub GUI consists of a badge list, an information tab, and a daily shop. This interface can be accessed directly from the menu or inside of any level, apart from cutscenes. Clicking on the "Information" button in the menu automatically brings up the information tab, allowing players to gain some information about the game. The badge list allows players to view descriptions of achievements they made and see the date of when they collected a badge. The information tab includes what a player's objective is, explanations of the game's features, a feedback system, and more. The shop tab allows players to purchase in-game cosmetics for coins that are sold in a shop that gets refreshed every 24 hours. It also includes a gamepass and an in-game currency section, allowing players to purchase upgrades and coins.

INSPECTING

When players collect certain artifacts or contain certain entities, they will be able to inspect them and see information like where they were found, the date they were collected, and a description of the item or entity that was collected. If a player hasn't yet contained or collected an artifact, question marks appear in place of each piece of information. Lock symbols also appear on uncontained entity tubes indicating that a specific entity still needs to be contained.

INTERACTING

Randomly-generated male and female NPCs roam around The Agency, allowing players to interact with them to get a random dialogue. Some of them also allow players to accept certain quests to complete for additional coins and XP. There are 5 categories of NPCs: Pistol, Rifle, Crate, Clipboard, and Radio. Clicking during a NPC's dialogue skips the typewriting animation to the end of the dialogue.

EARLY DEVELOPMENT

Back in 2018 and 2019, mysterious games known as "Roblox Myths" were an extremely popular subgenre on Roblox. Myth hunters loved to solve puzzles and unravel the hidden stories behind the creators of these places. I enjoyed exploring myth games myself, so I was motivated to make my own after discovering one of my accounts with the name StrandedBeneath.

❮
❯
1/5
I ended up making about a dozen of creepy places surrounded by this persona in order to attract players that loved a good mystery. The main game consisted of a log cabin in the middle of some snowy woods with a few puzzles that players could go around and solve. Although the game was extremely basic, especially for a myth game, I had a lot of fun messing around with all sorts of horror elements.

While the myth games I made were never extremely popular, they did gain some traction in the myth hunting community. One myth hunter named Detective_Phillips documented a few clues in a game of his (P&B Private Investigation Services) that were related to the myth games that I made. Another hunter named I_nsomniac drew some fan art of the persona I made. It was little instances like these that made me want to expand on this idea and create a fully fleshed out horror experience and see what else I was capable of creating.

❮
❯
1/3
Over the next few years, I came up with a few rough designs of what I wanted my horror game to look like. I still wanted to include a lot of the same elements as my original myth games, such as the masked persona and the black and white saturation, but I also wanted to create something that would appeal to those that aren't very familiar with Roblox myths. I wanted to create a more immersive experience for people. However, I still wanted to allow for a sense of mystery to the story. Ultimately, I wanted to create something that was interesting but also did justice to my original ideas.

❮
❯
1/4
After deciding to expand my myth games into a horror game, I had the opportunity to practice more things like character and scene design. The Watcher and Queen Azalea were the first entities I made for Stranded, and some of their earliest designs can be seen in the images on the right. I hope that one day players will grow to love and resonate with the characters that were made for this game.

❮
❯
1/3
That's about it for Stranded. This project is my proudest creation, as it has allowed me to grow as a designer, builder, programmer, and storyteller. I have big plans for this game despite a lot of it still being incomplete. I will be forever grateful to my friends and fans that have supported me over the years, especially those who have watched me give up on old projects to pursue this one.
